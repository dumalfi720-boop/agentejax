TOPIC 1

💬 Telegram Bot

The channel through which you will speak to your agent is aTelegram bot. Bot is an automatic account — you don't log in manually, your code controls it. Creation takes 2 minutes and the result is a token, which is the key that gives your program permission to read and send messages.

🎯How to create in 5 steps

  1. 1.On Telegram, open a conversation with@BotFather(the official bot for creating other bots).
  2. 2.Send/newbot.
  3. 3.Choose a display name (any) and a username ending in_bot.
  4. 4.Copy thetokenthat he sends (similar to123456:ABC...).
  5. 5.Send/startpro@userinfobotand copy your own chat ID — you will use it to release only you.

💡Practical tip

The token is equivalent to the bot's password. If it leaks, anyone controls their agent. Treat it like a bank password: never commit, never paste in public chat, and if it's leaked, ask for a new one on BotFather (/revoke).

✓ Do

  • ✓Restrict by chat ID (only you speak)
  • ✓Save token only in .env
  • ✓Use polling in dev, webhook in prod

✗ Avoid

  • ✗Token hardcoded in code
  • ✗Public bot without allowlist (anyone calls)
  • ✗Share BotFather screenshot
TOPIC 2
2

🔑 Model Key (LLM)

The model key is what gives your code permission to call the AI. Without it, the agent does not think. You havefour pathsand the best strategy is to use more than one.

🎯The 4 paths

  • Direct Anthropic:high quality, average price. Get the key at console.anthropic.com.
  • Direct OpenAI:high quality, mature ecosystem. platform.openai.com.
  • OpenRouter:just one key, dozens of models, includes DeepSeek and free models. openrouter.ai.
  • Local Hello:model running on your machine. Zero cost, but needs a decent GPU. ollama.ai.

📊Cost range (May 2026)

  • ~$0.003/thousand tokens inputDeepSeek and free templates on OpenRouter
  • ~$3/million tokens input· Claude Sonnet, GPT-4 family
  • ~$15/million tokens input· Premium templates (Claude Opus, GPT top)
  • $0· Local ollama with Llama 3.1, Qwen 2.5 or Mistral

💡Recommended strategy

Start with OpenRouter — one key, multiple models. Use DeepSeek or Llama 3.1 8B for simple tasks (cents a day), switch to Claude Sonnet for complex reasoning. In 2 minutes you only change the model string in the .env and the rest remains the same.

TOPIC 3
3

📦 Project Bootstrap

Here you create the skeleton:Node 20+, TypeScript, package.json and tsconfig.json. If this is already familiar to you, it's 3 minutes. If it isn't, read it calmly — this is the land where everything else will live.

🎯Command sequence

  1. 1.Check the Node:node -v(if < 20, update via nvm).
  2. 2.Create the project:mkdir agente && cd agente && npm init -y
  3. 3.Add TypeScript and tsx (executor):npm i -D typescript tsx @types/node
  4. 4.Create tsconfig withnpx tsc --initand adjust target ES2022, module Node16.
  5. 5.In package.json add"dev": "tsx watch src/index.ts"in the scripts.

💡Why tsx and not ts-node

tsx is faster, handles ES modules without a headache, has a built-in watch mode. It will save you 30 minutes of configuration that you would spend with alternatives.

TOPIC 4
4

🔐 Environment variables

Telegram token, LLM key, authorized IDs — these are allsecrets. They are kept in a file.envthat you never commit. In parallel, you create a.env.examplewith names (without values) to document which variables exist.

📝Minimal .env structure

TELEGRAM_BOT_TOKEN=123456:ABC-seu-token-real
ALLOWED_CHAT_IDS=123456789
LLM_PROVIDER=openrouter
LLM_API_KEY=sk-or-v1-...
LLM_MODEL=anthropic/claude-3.5-sonnet
DATABASE_PATH=./data/agente.db
TIMEZONE=America/Sao_Paulo

⚠️Attention

Before the firstgit add, confirm that the.gitignorehas the line.env. Token leaked on GitHub is discovered by bots in minutes and used for spam — it becomes your problem.

📁 Folder structure

Simple and clear convention: code goes insrc/, real-time data indata/, utility scripts inscripts/. Inside src, divide bydomain, not by type.

🌳Suggested layout

agente/
├── src/
│   ├── index.ts          # ponto de entrada
│   ├── config.ts         # carrega .env
│   ├── memory/           # camadas de memória
│   ├── llm/              # adapters de modelo
│   ├── tools/            # ferramentas (web, files, etc)
│   ├── telegram/         # bot e handlers
│   └── agent.ts          # loop principal
├── scripts/              # migrations, seeds, utilitários
├── data/                 # SQLite, logs (gitignored)
├── tests/
├── .env                  # SEGREDOS (gitignored)
├── .env.example          # template, sem valores
├── .gitignore
├── package.json
└── tsconfig.json

💡Tip

Each folder withinsrc/has aindex.tsthat exports what is public. The rest remains private. This creates a clear boundary between "internal use of the module" and "API for the rest of the project".

TOPIC 6
6

👋 First "hello agent"

Before building loop, memory or tools, write a scriptsmoke test: it only proves that Telegram, LLM and your code talk to each other. If this works, you're ready to scale.

🎯What does the smoke test do?

  1. 1.Reads the token and key from .env via dotenv.
  2. 2.Connect to Telegram and listen to a single message.
  3. 3.Send this message to LLM with a fixed system prompt ("Respond in one sentence").
  4. 4.Take the answer and send it back in the chat.
  5. 5.Closes.

If the answer arrives on your Telegram, everything is connected. If something fails, you know exactly which of the 4 layers (env, Telegram, LLM, code) to fix.

🔍Common mistakes in this step

  • LLM 401:wrong key or model that does not exist for this key.
  • Bot does not respond:wrong token, or you didn't send /start to the bot before.
  • "Cannot find module":forgot npm install after adding dependency.
  • Truncated answer:max_tokens too low in LLM payload.
